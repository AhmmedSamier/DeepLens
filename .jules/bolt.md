## 2024-05-24 - Pre-computed Lowercase Strings in Hot Paths
**Learning:** Calling `.toLowerCase()` inside a high-frequency loop (like scoring thousands of search items) creates significant overhead due to constant string allocation. However, if the codebase already utilizes a fuzzy-finding library (like Fuzzysort) with parallel pre-computed data structures, those structures often contain the lowercased strings already cached.
**Action:** In `SearchEngine` scoring loops, avoid calling `.toLowerCase()` on item properties. Instead, retrieve pre-computed lowercased strings from parallel Fuzzysort prepared arrays (e.g., `this.preparedFullNames[i]`) using `_targetLower` to eliminate redundant string allocations.
